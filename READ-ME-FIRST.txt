TRUSTOS — secure-demo-v1
=========================

This branch adds the executable /demo and removes every credential from
the codebase. No secret values appear anywhere in these files.


IMPORTANT — ROTATE YOUR SECRETS
-------------------------------
Any AUTH_SECRET value shared earlier in chat must be considered
compromised. Before deploying:

1. Generate a NEW AUTH_SECRET. On a Mac/Linux terminal:
      openssl rand -base64 32
   On Windows PowerShell:
      [Convert]::ToBase64String((1..32 | ForEach-Object {Get-Random -Max 256}))

2. If your Neon DATABASE_URL was ever pasted anywhere, reset the
   password in the Neon console (Roles -> reset password) and use the
   new connection string.

3. Put the NEW values only in Netlify -> Site configuration ->
   Environment variables. Never in a file, never in chat.


ENVIRONMENT VARIABLES
---------------------
See .env.example for the full list with EMPTY placeholders.
Required in Netlify:  DATABASE_URL, AUTH_SECRET, APP_URL
Optional (card payments): STRIPE_SECRET_KEY,
   NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY, STRIPE_WEBHOOK_SECRET
Seed-only (terminal, not Netlify): SEED_ADMIN_EMAIL, SEED_ADMIN_PASSWORD,
   SEED_VENDOR_EMAIL, SEED_VENDOR_PASSWORD


DEPLOY (GitHub Desktop)
-----------------------
1. GitHub Desktop -> Branch -> New branch -> "secure-demo-v1"
2. Show in Explorer. Delete everything EXCEPT the hidden ".git" folder.
3. Copy everything from inside this folder into the repo folder.
4. Commit, then Push. Netlify builds a preview for the branch.


DATABASE SETUP (once, on your laptop)
-------------------------------------
Set the env vars in your terminal, then:
   npm install
   npx prisma migrate deploy
   npm run db:seed
(The seed is idempotent — safe to run more than once.)


/demo
-----
Two allowlisted fake projects: Mini Momentz motherhood, Agara Live
stream. Buttons: Open as Vendor / Open as Client / Reset Demo.
Server resolves fixed demo identifiers from an allowlist; the endpoints
never accept an arbitrary projectId or role from the browser.


WHAT IS AND ISN'T BUILT
-----------------------
Live + tested: questionnaire, proposal, contract (with signature
evidence), payment (server-computed; Stripe paid only via webhook;
manual is a separate vendor-only endpoint labelled "not independently
verified"), milestones, approvals, completion, review, gallery/recording
link.
Not built: mood board, preparation/outfit guides, and dedicated
internet/equipment/stream-test screens (these exist as milestones).
